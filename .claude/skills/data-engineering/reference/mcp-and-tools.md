# 도구·MCP — 권장 오픈소스와 프로젝트 스코프 셋업 (문서화만)

> 예시 스택: Postgres + dbt. 원칙은 스택 불문 — 네 스택에 맞춰 적용.
> (어떤 DB·웨어하우스든 동일 원칙: 도구는 소비 프로젝트에 project-scope로만 배선하고,
> 자격증명은 환경변수로, 설치는 사용자 승인 후에. 도구 이름만 스택에 맞춘다.)

---

## 이 스킬이 잡아야 할 대표 상황

| 상황 | 무엇을 검토하나 |
|---|---|
| **에이전트가 DB 스키마를 직접 조회/검사하면 작업이 빨라질 때** | postgres-mcp로 스키마·인덱스·실행계획을 읽게 한다(읽기 권한 우선). |
| **dbt 모델·테스트·계보를 에이전트가 다루어야 할 때** | dbt MCP로 모델 실행·테스트·문서·계보를 노출한다. |
| **마이그레이션/쿼리 SQL을 정적으로 점검·방언 변환해야 할 때** | sqlglot로 파싱·검증·트랜스파일(코드 의존성, MCP 아님). |

**이 파일은 문서화만 한다 — 실제 배선(wiring)을 하지 않는다.** 이 프레임워크 레포에는 어떤 도구·MCP도 설치하지 않으며, **`.mcp.json`을 만들지 않는다**. 아래는 *소비 프로젝트*에서 필요할 때 사용자에게 권하기 위한 참조다.

---

## 0. 설치 규칙 — 절대 원칙

이 규칙을 어기면 도구를 추천하지 않는 것만 못하다.

- **글로벌 설치 금지.** `~/.claude/`·`~/.claude.json`·`claude mcp add --scope user`(또는 `global`) 같은 **글로벌** 스코프에 MCP·도구를 절대 설치하지 않는다. 글로벌 설치는 이 머신의 모든 프로젝트를 오염시키고, 무관한 프로젝트에서 DB에 연결되는 사고를 일으킨다.
- **project-scope로만.** 소비 프로젝트의 저장소 안(`<repo>/.mcp.json`) 또는 `claude mcp add <name> --scope project`(또는 `local`)로만 배선한다. scope를 **항상 명시**한다.
- **사용자 승인 후에만.** 무엇을·왜·어떻게 배선하는지 한국어로 설명하고, 사용자가 **그 항목을 명시적으로 승인**한 뒤에 설치한다. "알아서 해줘" 같은 포괄 허락은 글로벌 설치 허가가 아니다.
- **자격증명은 환경변수로.** 연결 문자열·비밀번호를 `.mcp.json`이나 코드에 하드코딩하지 않는다. `${ENV_VAR}` 참조로 두고 실제 값은 `.env`(gitignore)·시크릿 매니저에.
- **읽기 권한 우선.** DB MCP는 가능하면 read-only 계정으로 붙인다. 쓰기·DDL은 사람이 검토한 마이그레이션 경로로만.

---

## 1. postgres-mcp — Postgres를 에이전트가 직접 조회

- 저장소: `github.com/crystaldba/postgres-mcp`
- 용도: 에이전트가 스키마·인덱스·제약을 조회하고, `EXPLAIN`으로 실행계획을 보고, (허용 시) 쿼리를 실행한다. 스키마 설계 리뷰·느린 쿼리 진단에 강력하다.
- 권장: **read-only DB 롤**로 연결. 운영 DB가 아니라 개발/스테이징/복제본에 붙인다.
- 배선(소비 프로젝트, 승인 후): `claude mcp add postgres --scope project ...` 또는 `<repo>/.mcp.json`에 등록, 연결 문자열은 환경변수.

---

## 2. dbt MCP — dbt 프로젝트를 에이전트가 운용

- 저장소: `github.com/dbt-labs/dbt-mcp`
- 용도: dbt 모델 실행·`dbt test`·문서·**계보(lineage)** 조회를 에이전트에 노출한다. 변환 모델 검증(`data-validation.md`의 dbt tests)과 함께 쓰면 좋다.
- 권장: dbt 프로젝트가 있는 소비 레포에서 project-scope로. 프로파일/자격증명은 환경변수·`profiles.yml`(gitignore)로.

---

## 3. sqlglot — SQL 파싱·검증·방언 변환 (코드 의존성)

- 저장소/패키지: `sqlglot` (`github.com/tobymao/sqlglot`)
- 용도: SQL을 파싱해 **정적 점검**, 방언 간 **트랜스파일**(예: 작성은 Postgres, 타깃은 BigQuery), 마이그레이션·쿼리의 구문/구조 검증. MCP가 아니라 라이브러리 의존성으로 들인다(`pip install sqlglot` 등 프로젝트 의존성에).
- 용처: 마이그레이션 SQL의 사전 린트, 쿼리 동등성 점검, 멀티 웨어하우스 지원 시 방언 변환.

---

## 4. 그 외 권장 오픈소스 (스택에 맞춰)

도구는 소비 프로젝트의 의존성으로 들이고, 위 설치 규칙(project-scope·승인·환경변수)을 동일하게 따른다.

| 영역 | 도구 | 비고 |
|---|---|---|
| 데이터 품질·기대치 | **Great Expectations**, **Soda** | 적재 게이트·리포트 |
| 데이터프레임 스키마 | **Pandera** | pandas/Spark 경계 검증 |
| 마이그레이션 | **Alembic**(SQLAlchemy) · **Flyway** · **Prisma Migrate** · Liquibase | 스택에 맞춰 하나 |
| 변환·모델링 | **dbt** | 분석·웨어하우스 ELT |
| 적재 파이프라인 | **dlt**, **Airbyte** | EL(추출·적재) |

---

## 5. 도구 도입 절차 (요약)

1. **필요 식별** — 어떤 작업이 왜 이 도구로 빨라지는지 한국어로 설명한다.
2. **scope 결정** — 항상 project(또는 local). 글로벌은 검토 대상이 아니다.
3. **사용자 승인** — 그 항목을 콕 집어 승인받는다.
4. **배선** — `<repo>/.mcp.json` 또는 `claude mcp add <name> --scope project`. 자격증명은 환경변수.
5. **권한 최소화** — DB는 read-only 우선, 쓰기는 검토된 마이그레이션 경로로.
6. **이 레포에는 설치하지 않는다** — 여긴 프레임워크다. `.mcp.json`을 만들지 않는다.
