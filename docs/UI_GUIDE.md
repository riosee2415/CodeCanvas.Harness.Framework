## AI 슬롭 안티패턴 — 하지 마라
| 금지 사항 | 이유 |
|-----------|------|
| backdrop-filter: blur() | glass morphism은 AI 템플릿의 가장 흔한 징후 |
| gradient-text (배경 그라데이션 텍스트) | AI가 만든 SaaS 랜딩의 1번 특징 |
| "Powered by AI" 배지 | 기능이 아니라 장식. 사용자에게 가치 없음 |
| box-shadow 글로우 애니메이션 | 네온 글로우 = AI 슬롭 |
| 보라/인디고 브랜드 색상 | "AI = 보라색" 클리셰 |
| 모든 카드에 동일한 rounded-2xl | 균일한 둥근 모서리는 템플릿 느낌 |
| 배경 gradient orb (blur-3xl 원형) | 모든 AI 랜딩 페이지에 있는 장식 |

## 컴포넌트

```

```

### 입력 필드
```
{예: rounded-lg bg-neutral-900 border border-neutral-800 px-4 py-3}
```

## 레이아웃
- 전체 너비: {예: max-w-5xl}
- 정렬: {예: 좌측 정렬 기본. 중앙 정렬 금지}
- 간격: {예: gap-3~4, 섹션 간 space-y-8}

## 간격 시스템
임의의 px 값을 쓰지 말고 정해진 스케일만 사용한다 (일관성).
- 스케일: {예: 4의 배수 — 4, 8, 12, 16, 24, 32, 48}
- 컴포넌트 내부 패딩: {예: p-4 / p-6}
- 요소 간 간격: {예: gap-2 / gap-3}

## 반응형
- 브레이크포인트: {예: 모바일 우선. sm:640 / md:768 / lg:1024}
- {예: 데스크탑 다단 → 모바일 단일 컬럼으로 스택}
- {예: 터치 타깃 최소 44×44px}

## 상태 표현
모든 데이터 영역은 4가지 상태를 빠짐없이 처리한다. (AI 산출물이 가장 흔히 빠뜨리는 부분.)
| 상태 | 처리 |
|------|------|
| 로딩(loading) | {예: 스켈레톤 or 인라인 스피너. 레이아웃 점프 금지} |
| 빈(empty) | {예: 안내 문구 + 다음 행동 유도. 빈 화면 금지} |
| 에러(error) | {예: 원인 + 재시도 경로 제시} |
| 정상(success) | {기본 렌더링} |

## 접근성 (a11y)
- 대비: {예: 본문 텍스트 대비 WCAG AA(4.5:1) 이상. neutral-500 위 작은 텍스트 주의}
- 포커스: {예: 모든 인터랙티브 요소에 보이는 focus-visible 링}
- 키보드: {예: 마우스 없이 모든 기능 도달 가능. 커스텀 위젯에 적절한 role/aria}
- 이미지/아이콘: {예: 의미 있는 아이콘에는 aria-label, 장식용은 aria-hidden}
